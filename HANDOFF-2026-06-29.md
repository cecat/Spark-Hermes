## Important findings worth carrying forward

### F-A — Don't suppress symptoms; chase root causes
The prior `HERMES_SUPPRESS_SETHOME_NOTICE` workaround masked the real bug
(undeclared slash command) for 9 days. The suppression patch itself didn't
even work (got wiped 90 min after every rebuild). Operator's pushback
("isn't this jury-rigged?") was the right call. Codified in the rewritten
memory `project_hermes_sethome_notice_suppressed.md`.

### F-B — Gateway integrity-hash drifts during normal use
The Hermes gateway writes to `/sandbox/.hermes/.env` at runtime (e.g.
`/sethome` updates `<PLATFORM>_HOME_CHANNEL`) but does **not** recompute
`/etc/nemoclaw/hermes.config-hash`. Result: every `/sethome` invocation
silently desyncs the hash file. The next `nemohermes gandalf recover` or
container restart will refuse to launch. **Always rehash after touching
.env or config.yaml.** The pattern is in `ops/post-rebuild.sh` section 2d.

### F-C — `ops/post-rebuild.sh` runtime patches don't survive container restarts
NemoClaw can restart the sandbox container for reasons we haven't fully
characterized (F-35). When that happens, the writable Docker layer is reset
and any in-container file edits (`/opt/hermes/...`) vanish. **Don't patch
files at runtime expecting them to persist.** If you genuinely need a
source patch, it has to go into the image build (Dockerfile) or be
re-applied on every gateway start (hook script).

### F-D — `response_url` POSTs to `hooks.slack.com` fail from the live gateway
Concrete reproduction: in the long-running gateway's aiohttp, POSTs to
`hooks.slack.com` fail with `Cannot connect to host hooks.slack.com:443
[Temporary failure in name resolution]`. From a **fresh** Python subprocess
inside the same sandbox, the same aiohttp call succeeds. DNS resolves fine
via `getent`. curl from the sandbox succeeds. Other Slack hosts work for
the gateway process. The misleading "name resolution" error is almost
certainly OpenShell L7 proxy rejecting the unauthenticated callback POST
because of `request_body_credential_rewrite: true` on the hooks.slack.com
endpoint in the built-in preset. **This blocks all Slack slash command
result delivery.** See F-30 below.

### F-E — Slash command ownership in Slack is workspace-wide
A slash command name like `/hermes` can be claimed by only one app in a
workspace. Re-installing your app with that command in its manifest
**reassigns ownership** to your app (Slack tells you this in the consent
flow). Worth knowing if multiple apps in the workspace declare overlapping
commands. In our case, `Somm` (some other TPC member's app) was the prior
owner; we now own `/hermes`. F-34 followup.

### F-F — Telegram allowlist env var is `TELEGRAM_ALLOWED_IDS`, not `_USERS`
Upstream NemoClaw source reads `TELEGRAM_ALLOWED_IDS`. An earlier draft of
the secrets template + README used `_USERS` (renamed today in
`bringup/secrets.example.env`). The live `~/.hermes/.env` also got renamed.
Anywhere else that says `_USERS` for Telegram is stale.

### F-G — `slack_home_channel` is `D0BBDHYCWPK` not `C0BAV5A4C7R`
The retired private channel `#agent-gandalf` (C0BAV5A4C7R) was retired
2026-06-18 in favor of the DM `D0BBDHYCWPK`. Earlier `~/.hermes/.env`
still pointed at the retired channel. Fixed in the host .env today.
`config.yaml` carries the correct value as `home_channel_id`.

## How to safely cycle the gateway

NemoClaw's `nemohermes gandalf recover` short-circuits if the gateway is
healthy — it only restarts an unhealthy one. To **force** a restart
(needed if you've edited `.env` and want the gateway to pick up new values):

```bash
CON=$(docker ps --format '{{.Names}}' | grep ^openshell-gandalf-)
docker exec -u root "$CON" kill -TERM $(docker exec -u root "$CON" pgrep -f 'hermes gateway run' | head -1)
# nemoclaw-start respawns it within ~5 seconds. PID gets reassigned (often to a low number).
```

**Always rehash before restarting** if you touched `.env` or `config.yaml`:

```bash
docker exec -u root "$CON" sh -c '
  cd /sandbox/.hermes &&
  cfg_hash=$(sha256sum config.yaml | awk "{print \$1}") &&
  env_hash=$(sha256sum .env | awk "{print \$1}") &&
  cat > /etc/nemoclaw/hermes.config-hash <<EOF
$cfg_hash  /sandbox/.hermes/config.yaml
$env_hash  /sandbox/.hermes/.env
EOF'
```

**Always snapshot first** for nontrivial changes (CLAUDE.md destructive-ops
policy):

```bash
nemohermes gandalf snapshot create --name pre-<descriptive>
```

(Note: snapshot CLI sometimes prints `Snapshot failed. Failed files:
runtime/state.db` but the snapshot itself succeeds — verify with
`nemohermes gandalf snapshot list`.)

## How to debug "slash command isn't working" in Slack

Diagnostic ladder (cheapest first):

1. Type `/` in the Slack DM. Does `/hermes` appear in the picker?
   - **No** → manifest issue or workspace install issue.
   - **Yes** → continue.
2. Type `/hermes help`. Does Slack show "Running /hermes..."?
   - **No, app did not respond** → Slack sent it but gateway didn't ack within 3s.
   - **Yes** → Slack delivered it to your app; result delivery is the problem.
3. Check the live gateway log for the attempt:
   ```bash
   CON=$(docker ps --format '{{.Names}}' | grep ^openshell-gandalf-)
   GW_PID=$(docker exec -u root "$CON" pgrep -f 'hermes gateway run' | head -1)
   # Note: when post-rebuild.sh runs, it can overwrite the on-disk log file
   # while the gateway holds the FD. Read via /proc to see live writes:
   docker exec -u root "$CON" sh -c "cat /proc/$GW_PID/fd/8" | tail -60
   # FD 8 is gateway.log; FD 1 is /tmp/gateway.log (stderr). Verify FD numbers
   # via `ls -la /proc/$GW_PID/fd/`.
   ```
4. Look for `response_url POST failed: ... hooks.slack.com` — that's F-30.
   Until F-30 is fixed, ANY Slack slash command appears to hang to the user,
   even though the action ran server-side. Workaround for `/sethome`: write
   the `*_HOME_CHANNEL` env var directly (see today's runlog).

## What NOT to do

- **Don't reintroduce `HERMES_SUPPRESS_SETHOME_NOTICE` or any `gateway/run.py`
  source patch.** The memory file
  `project_hermes_sethome_notice_suppressed.md` explicitly warns against this.
- **Don't restart the container** (`docker restart`) to fix gateway-process
  issues. SIGTERM the gateway PID instead; nemoclaw-start respawns it. A full
  container restart loses Tavily key, sethome envs, and other runtime-injected
  state until post-rebuild.sh runs.
- **Don't edit `/sandbox/.hermes/config.yaml` directly** unless you also
  rehash. The gateway will refuse to start.
- **Don't run `nemohermes gandalf channels add|remove` casually** — they
  trigger a sandbox rebuild. CLAUDE.md destructive-ops policy applies.
- **Don't trust the `Snapshot failed. Failed files: runtime/state.db`
  message.** The snapshot usually succeeded anyway. Verify via `snapshot list`.
